# Hook: `experimental.chat.system.transform`

> **Status: EXPERIMENTAL** — This hook's signature and behavior may change between opencode versions without a semver major bump.

## Trigger

Fires during LLM request preparation, after the base system prompt array is assembled but before it is finalized. Also fires during agent metadata generation (title/description generation for agents). Fires on every LLM call.

## Call Sites

- `packages/opencode/src/session/llm/request.ts` — main LLM request preparation.
- `packages/opencode/src/agent/agent.ts` — agent title/description generation.

## Input Type

```typescript
{
  sessionID?: string;
  model: Model;
}
```

- `sessionID` — the active session ID. May be **absent** during agent metadata generation (title/description generation calls that are not associated with a user-facing session).
- `model` — the resolved `Model` object for the LLM call being prepared.

## Output Type

Mutate in place:

```typescript
{
  system: string[];
}
```

- `system` — array of system prompt strings. The final system prompt sent to the provider is either the joined array or each element sent as a separate system message, depending on provider capabilities. Mutate by pushing, splicing, or replacing elements.

### System prompt structure

- `system[0]` — the core "header" system prompt generated by opencode. Contains agent instructions, tool guidance, and context.
- `system[1...]` — additional system prompt sections appended after the header.

## Signature

```typescript
"experimental.chat.system.transform"?: (input: {
  sessionID?: string;
  model: Model;
}, output: {
  system: string[];
}) => Promise<void>
```

## Execution Model

- Called sequentially across all loaded plugins, in load order.
- Hook is async-awaited.
- Throwing from this hook propagates through `Effect.promise()` — the LLM call fails.
- Multiple plugins each mutate `output.system` in sequence; each sees the state left by the previous plugin.

## Notes

- The most common use is `output.system.push(myInstructions)` — appending a new section without touching the existing prompt.
- Replacing `output.system[0]` (the core header) discards all built-in opencode instructions. This is rarely correct; prefer appending.
- When `input.sessionID` is absent, the call is for agent metadata generation — a lightweight internal call. Guard accordingly if your additions are session-specific.
- Use `input.model` to apply system prompt modifications conditionally per model or provider (e.g., inject different instructions for reasoning models vs. standard models).
- System prompt additions here affect every LLM call, including subagent calls. They are not scoped to user-visible sessions unless you check `input.sessionID`.
- The hook name prefix `experimental.` signals this hook is subject to change. Pin your plugin to a specific opencode version if you depend on it.

## Example

```typescript
import type { Plugin } from "@opencode-ai/plugin"

export const SystemPromptPlugin: Plugin = async (ctx, options) => {
  const companyName = options?.companyName as string ?? "Acme Corp"

  return {
    "experimental.chat.system.transform": async (input, output) => {
      if (!input.sessionID) return  // skip agent metadata generation calls

      output.system.push(
        `You are operating within the ${companyName} development environment. ` +
        `All code changes must follow the company coding standards. ` +
        `Do not commit directly to the main branch.`
      )
    },
  }
}
```
